FindeX is a precision-engineered file discovery engine for Windows. No indexing, no background data collection—just pure speed.

The core principles.

Modern computing has become synonymous with waiting. We wait for cloud sync, we wait for indexing, and we wait for background processes. FindeX was built to eliminate that friction.

By leveraging direct file system queries and high-concurrency search algorithms, FindeX provides a surgical tool for finding assets, code, and documents without the overhead of the standard OS search features.

Non-Indexing

We believe in "Live Search." Your results are always current because they are read directly from the disk at the moment of request.

Modern Aesthetics

Built using the Mica design language, FindeX feels like a native part of the Windows 11 ecosystem.

Power in every query.

Advanced Parameters

Apply constraints based on file size, creation date, and metadata without complex syntax.

Scoped Context

Limit searches to specific project folders or expand to entire drives with a single toggle.

Native Interactions

Standard Windows context menus are built directly into the result list for seamless workflow.

FindeX Documentation

Introduction

This guide covers the installation, configuration, and advanced usage of FindeX. Whether you are using the pre-compiled binary or building from source, follow the instructions below to get started.

Installation Guide

FindeX is designed to be portable.

  • Download the FindeX.exe from the Downloads section. Place it in any folder and run it.
  • You can download the source code from our offical GitHub page, and run python FindeX/main.py.

Advanced Search Syntax

While standard search is intuitive, you can use Regex Search for refined results:

ext:pdf — Only find PDF files.

size:>500mb — Find files larger than 500 megabytes.

date:today — Find files modified in the last 24 hours.

Technical Architecture

FindeX operates on a multi-threaded architecture to ensure the UI remains responsive even during heavy disk I/O operations.

Search Pipeline:
1. Input Capture (Main Thread)
2. Task Delegation (Worker Thread Pool)
3. OS Query (Direct System API Calls)
4. UI Update (Callback to Main Thread)
